Welcome

This sample project shows how to use the Cloudinary Node.js SDK to upload files to your product environment in various ways:

Uploading from the CLI

There are several use-cases for uploading files from the filesystem using the Node.js SDK via the command line. To showcase such features there are two API endpoints that can be leveraged, these are:

  • /upload-from-local - uploads the two files that are bundled with this repository /photos/singapore.jpeg and photos/sydney.jpeg.
  • /upload-large-from-local - uploads a video from a folder that is bundled with this repository video/singapore.mp4

Uploading from the browser

The rest of the API endpoints are handling file uploads from the browser using various strategies.

  • Upload from browser : Uses the cloudinary.uploader.upload_chunked_stream method to upload a file, which has been converted to a buffer.
  • Upload large files : Uses the cloudinary.uploader.upload_large method to upload a large file in chunks. This is useful for managing large file uploads efficiently, especially in cases where the client may be constrained by bandwidth or where interruptions could occur during the upload.
  • Upload large files via stream : Uses the cloudinary.uploader.upload_stream method to upload a large file using a stream.

Each of the upload methods also applies the tag, nodejs-sample, to the uploaded asset. This tag is used to retrieve all the media uploaded through this app, which you can view in the Gallery.
